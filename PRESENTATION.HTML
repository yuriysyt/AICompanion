AI Companion for the
Visually Impaired

A Local-First, Agentic AI for Physical OS Interaction

Interdisciplinary Project Design (IPD)

The Accessibility Gap

  • UI/Cursor-Centric Systems: Modern Operating Systems (Windows, macOS) demand high visual acuity and precise motor control, excluding millions of users.
  • Reactive, Not Active: Existing accessibility tools like NVDA only read the screen back to users. They do NOT take physical actions on the user's behalf.
  • API-Bound Limitations: Voice assistants (Siri, Windows Copilot) cannot physically control generic desktop software; they are restricted to isolated API tasks.
  • The Privacy Threat: Cloud-based AI systems require sending screen data and private user documents to external servers (OpenAI/Google), creating unacceptable privacy risks.

Project Vision & Strategy

Local-First

100% on-device execution. We utilize local LLMs via Ollama to guarantee absolute data privacy. No user documents or screen captures ever leave the physical machine.

Agentic Action

Translating human intent into physical operations. The AI dynamically generates sequential plans, moving the mouse and typing exactly as a human operator would.

Voice-Driven

Frictionless UX completely bypassing traditional UI navigation. Audio inputs are parsed, contextualized, and executed with near-zero latency.

High-Level Microservice Architecture

Frontend Engine (C# WPF .NET 8)

  • Responsibility: Bare-metal OS interaction, UI rendering, and voice capture.
  • Performance: Compiled, low-latency execution utilizing direct Win32 API hooks.
  • Task Routing: Intercepts and filters simple commands locally without invoking the AI engine.
  • UX Integrity: Modern, accessible MVVM architecture (`MainWindow.xaml` & ViewModels).

AI Brain (Python FastAPI)

  • Responsibility: Deep reasoning, intent extraction, memory, and multi-step JSON planning.
  • Ecosystem: Leverages the mature Python AI tooling landscape.
  • Endpoints: Exposes asynchronous REST endpoints (`/api/plan`, `/api/intent`).
  • Scalability: Decoupled nature allows swapping out underlying LLMs seamlessly.

The C# Execution Engine

  • The Gateway: LocalCommandProcessor.cs strips conversational filler ("please", "can you") from inputs.
  • Zero-Latency Routing: Uses local Regex and FuzzySharp (Levenshtein distance) to instantly execute simple commands (e.g., "open calculator") without network inference.
  • The Muscle: AgenticExecutionService.cs receives JSON action plans from the Python backend.
  • Bare-Metal Control: Executes plans using raw Win32 APIs for precise physical keystrokes and UI interactions.

The Python AI Brain

  • The LLM Engine: Powered by IBM Granite 3-dense, an enterprise-grade model optimized for structured reasoning.
  • Local Hosting: Run 100% locally via Ollama, completely severing reliance on cloud compute.
  • The API Bridge: server.py and agentic_planner.py manage REST requests from the C# frontend.
  • Task Breakdown: Converts ambiguous user speech ("Open Notepad, write a grocery list, then save it") into structured, actionable JSON arrays.

Vector Memory & Context

  • Embedded Database: We integrate ChromaDB directly into the Python microservice.
  • Dialogue History: Stores the most recent conversational history (up to 10 dialogue turns).
  • Contextual Awareness: Allows the AI to understand pronouns and references (e.g., "Close it", "Read the second line again").
  • State Persistence: Prevents the "amnesia" common in stateless API architectures, crucial for complex, multi-step accessibility tasks.

Voice Processing Reliability

  • UnifiedVoiceManager.cs Intercepts microphone audio at the OS level, managing streams asynchronously to prevent UI thread blocking.
  • Premium Primary Pipeline Primary Text-to-Speech (TTS) and Speech-to-Text (STT) are powered by the ElevenLabs premium cloud API for ultra-realistic synthesis.
  • Offline Whisper.net Fallback If internet connectivity drops, the system seamlessly hot-swaps to an offline ggml-tiny.en.bin Whisper model without failing the user session.

The Agentic Planner Pipeline

1. Voice Input

User speaks: "Open Notepad, write a grocery list, then save it."

2. Intent Extraction

IBM Granite processes text, identifies multi-step OS requirements.

3. JSON Generation

[{"action": "open_app", "target": "notepad"}, {"action": "type", "target": "milk"}]

4. Win32 Execution

C# AgenticExecutionService parses JSON, triggers OS physical inputs.

Win32 Target Safety & Focus

  • The Rogue Keystroke Problem: Blindly sending keystrokes can result in typing passwords into public chat windows if focus shifts unexpectedly.
  • Focus Verification: The C# engine strictly verifies window handles (`HWND`) before every single simulated physical action.
  • API Utilization: Uses SetForegroundWindow and AttachThreadInput to forcefully seize and lock target contexts.
  • Fail-Safe Halting: If the target window unexpectedly loses focus, the execution pipeline instantly halts, protecting user data.

Project Evolution

Phase 1

The C# Monolith: Standalone app using fragile SendKeys and local regex. High failure rate on complex tasks.

Phase 2-3

Python API & Planner: Split architecture. Integrated OpenClaw principles and IBM Granite JSON task generation.

Phase 4-5

Security & Refactor: Rewrote C# engine with robust Win32 GUI handles. Fixed context loss and ChromaDB memory bugs.

Phase 6

QA & Speed: Reduced execution latency, added shared HTTP clients, implemented offline Whisper fallback.

Roadmap (Phase 7): True Vision

  • Bypassing API Limits: Moving from "blind Win32 APIs" to true pixel-level computer vision control.
  • Base64 Screen Capture: The C# app will capture screen states and push Base64 frames to the Python Vision AI.
  • Spatial Mapping: The AI will "look" at the screen and return exact (X, Y) pixel coordinates of target buttons.
  • Physical Mouse Actuation: C# will move the physical mouse cursor to the mapped coordinates, interacting with non-standard UIs flawlessly.

Roadmap: Conversational AI & Security

Interactive Two-Way Dialogue

  • Shifting the AI from a silent "Do-er" to an active participant.
  • Clarification Prompts: "I see two documents named 'Tax'. Which one should I open?"
  • Dynamic re-planning based on mid-execution user corrections.
  • Creates a genuine "companion" UX rather than a rigid command line.

Secure PIN Authorization

  • Critical safety barrier for destructive or sensitive operations.
  • Applies to: Sending emails, deleting files, purchasing, banking.
  • OS Halting: Execution suspends until user verbally provides a predefined Secure PIN.
  • Prevents accidental triggers and malicious prompt injections.

Roadmap: Advanced Disability Suite

  • SQLite Multi-User Database Registration system to store personalized accessibility settings, disabled profiles, and distinct ChromaDB memory banks per user.
  • Voice-Activated Magnification Integration with OS-level magnifier APIs to zoom into specific UI quadrants strictly via voice command.
  • Cursor OCR & Element Narration Real-time Optical Character Recognition (OCR) under the mouse cursor, narrating screen elements to assist partially-sighted users navigating complex UIs.
